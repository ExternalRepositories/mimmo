Detailed Description

MultipleMimmoGeometries is an executable block class capable of I/O hanfling of multiple geometries files of the same topology.

MultipleMultipleMimmoGeometries is the object to manage the reading/writing of one or more geometry files of the same topology to/from a list of standard MimmoObjects. It works as a MimmoGeometry class, and retains almost the same features. The meshes to import/export have to be meshes with constant type elements. The valid format are: binary .stl, ascii .vtu (triangle/quadrilateral elements) and ascii .nas (triangle elements) for surface mesh; ascii .vtu (tetra/hexa elements) for volume mesh. ofp or pcvtu point cloud formats for generic point cloud structures. The current class can be used in Reading or Writing modes once, not both at the same time.


It uses smart enums FileType list of available geometry formats, which are:

  • STL = 0 Ascii/Binary triangulation stl.
  • STVTU = 1 Surface triangulation vtu.
  • SQVTU = 2 Surface quadrilateral vtu.
  • VTVTU = 3 Volume tetrahedral VTU.
  • VHVTU = 4 Volume hexahedral VTU.
  • NAS = 5 Nastran triangulation nas.
  • OFP = 6 Ascii OpenFoam point cloud.
  • PCVTU = 7 Point Cloud VTU
  • CURVEVTU= 8 3D Curve in VTU

Outside this list of options, the class cannot hold any other type of formats for now. The smart enum can be recalled in every moment in your code, just using mimmo::FileType and including MultipleMimmoGeometries header.
